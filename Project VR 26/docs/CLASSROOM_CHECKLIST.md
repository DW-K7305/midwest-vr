# Classroom checklist — onboarding new headsets

Print this and tick boxes as you work through a fresh Quest 2.
Allow ~90 seconds per headset.

---

## Per Meta account (do this once total)

- [ ] On the phone, open **Meta Quest** app.
- [ ] **Menu → Devices → Headset settings → Developer mode → Create developer
      organization** (any name works, e.g. "Classroom").
- [ ] Accept the developer terms.

---

## Per headset (every Quest 2 you want to manage)

### On the phone
- [ ] Open **Meta Quest** app.
- [ ] **Menu → Devices** → tap the headset.
- [ ] **Headset settings → Developer mode → On**.

### On the Mac
- [ ] Plug headset into Mac with the USB-C cable from its box.
- [ ] Open **MidWest-VR**.

### On the headset (put it on)
- [ ] When the **Allow USB debugging?** dialog appears:
  - [ ] Tick **Always allow from this computer**.
  - [ ] Tap **OK**.

### Verify
- [ ] On the Mac, MidWest-VR Dashboard shows the headset with a green
      **Online** badge.
- [ ] Battery percentage and storage are visible.

### Configure (optional but recommended for student stations)
- [ ] **Wi-Fi page** → enter classroom SSID + password → **Send to headset**
      → finish join inside the headset.
- [ ] **Apps page** → **Install APK…** → pick the curriculum app → wait for
      the install progress to finish.
- [ ] **Kiosk page** → tap **Pin to this app** for the curriculum app.
      The headset will boot straight into that app from now on.

### Label
- [ ] Put a sticker on the headset with its serial number's last 4 digits
      (visible in MidWest-VR's device card) so you can find it again quickly.

---

## When something's wrong

- **Card says "Tap Allow on headset"** — put the headset on; the dialog is
  waiting.
- **No card at all** — wrong cable. Use the headset's box cable or a known
  USB-C *data* cable.
- **Card flickers Online/Offline** — bad cable or port. Swap one or the other.
- **Developer Mode toggle missing** — the Meta account isn't a dev org member
  yet. See the per-account section above.

For deeper troubleshooting see `docs/CONNECT_HEADSETS.md`.
